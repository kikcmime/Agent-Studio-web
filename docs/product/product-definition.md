# Agent Studio 产品定义

## 1. 产品一句话

Agent Studio 是一个面向开发者和 AI 应用搭建者的可视化多 Agent 编排平台。

第一版目标不是做成完整的 Dify 替代品，而是做一个可以稳定承载以下核心能力的工作台：

- 用画布定义一个 Flow
- 在 Flow 中挂载多个 Agent
- 给 Agent 绑定模型、Skill、MCP 工具和知识库
- 执行一次 Flow，并看到运行轨迹、输入输出和节点状态

## 2. 为什么做这个

这个项目的学习目标不是单独练前端或单独练后端，而是通过一个完整产品把以下能力串起来：

- 前端：复杂界面建模、画布交互、状态管理、组件抽象
- 后端：工作流编排、任务执行、Agent 运行时、数据建模
- 工程：兼容性设计、模块边界、版本演进、逐步迭代

所以产品必须满足两个要求：

- 第一版足够小，能做完
- 架构足够稳，后续能继续长

## 3. 第一版做什么

第一版只做一个核心场景：

用户创建一个 Flow，在画布中放入开始节点、Agent 节点、条件节点和结束节点；每个 Agent 节点可以绑定一个 Agent 配置；Agent 配置可以选择模型、挂接 Skill、MCP 工具和知识库；点击运行后，系统按流程执行，并展示每个节点的输入、输出、状态和日志。

## 4. 第一版不做什么

为了保证能落地，以下内容不进入 v1：

- 多租户权限系统
- 在线协同编辑
- 完整插件市场
- 复杂审批/发布流程
- 训练、微调、评测平台
- 非结构化超复杂知识库加工流水线
- 超多节点类型

## 5. 产品核心对象

第一版只围绕 6 个对象展开：

### 5.1 Flow

一个可视化工作流定义，包含节点、边、版本和发布状态。

### 5.2 Agent

一个可复用的执行单元，定义角色、提示词、模型配置、默认上下文和可用能力。

### 5.3 Skill

一个结构化能力描述，代表一组稳定的调用能力或工具编排。

在 v1 中，Skill 先作为“可绑定的能力描述”存在，不追求复杂运行机制。

### 5.4 MCP Server / Tool

外部能力接入层。v1 只要求支持注册、启用、为 Agent 绑定，不要求做全量生态兼容。

### 5.5 Knowledge Base

给 Agent 提供检索上下文的数据源。v1 先支持最小闭环：

- 知识库元信息
- 文档上传记录
- 切片与索引任务占位
- Agent 绑定关系

### 5.6 Run

一次 Flow 执行实例，保存整体状态、节点执行轨迹、日志、耗时和结果。

## 6. 用户视角的最小闭环

第一版用户完整路径如下：

1. 创建一个 Agent
2. 给 Agent 绑定模型、Skill、MCP、知识库
3. 创建一个 Flow
4. 在画布中编排节点
5. 将 Agent 节点绑定到已有 Agent
6. 运行 Flow
7. 查看运行轨迹和节点结果

只要这 7 步打通，产品就成立。

## 7. 页面范围

第一版前端页面只保留 5 个：

- Flow 列表页
- Flow 画布页
- Agent 列表页
- Agent 配置页
- Run 详情页

知识库、Skill、MCP 暂时不单独做复杂页面，可以先以内嵌抽屉或简单管理页存在。

## 8. 设计原则

### 8.1 Flow 和 Agent 解耦

Flow 负责“怎么编排”。
Agent 负责“怎么执行”。

两者不要揉在一起，否则后面很难复用。

### 8.2 配置先行，执行后置

先把配置模型定义清楚，再做执行引擎。
第一版允许部分能力先是配置占位，但数据结构必须稳定。

### 8.3 统一资源绑定方式

Skill、MCP、知识库都作为 Agent 的“能力绑定资源”对待，避免每类资源有一套完全不同的接口。

### 8.4 先串行，后并行

v1 的 Flow 执行以串行和简单分支为主。
并行执行、回溯、重试编排放到后续版本。

## 9. 版本路线

### v1

- 单工作区
- Flow 画布编排
- Agent 绑定能力资源
- 串行执行和简单条件分支
- Run 轨迹查看

### v1.5

- Agent 模板
- Flow 版本快照
- 知识库基础检索
- 节点级重试

### v2

- 并行节点
- 子流程
- 发布环境
- 更完整的 Skill / MCP 生命周期管理

## 10. 我们现在要做的东西

我们现在不是做一个“什么都能配”的平台。

我们要做的是：

一个面向开发者的 Agent Studio 基础版，让用户能通过画布编排多个 Agent，并把模型、Skill、MCP、知识库绑定到 Agent 上完成一次真实可观测的运行。
